# 预设

## 共享依赖

适配器仍然会自动共享常见的 Next 内部依赖，通常不需要重复声明：

- `next/dynamic`
- `next/head`
- `next/image`
- `next/link`
- `next/router`
- `next/script`
- `react`
- `react/`
- `react-dom`
- `react-dom/`
- `styled-jsx`
- `styled-jsx/style`

## Pages 暴露

新的适配器 **不再支持** `extraOptions.exposePages`。

如果另一个应用需要导入某个拼接后的 Pages 路由，请显式声明它：

```js
new NextFederationPlugin({
  name: 'remoteApp',
  filename: 'static/chunks/remoteEntry.js',
  exposes: {
    './pages/index': './pages/index',
    './pages/shop/products/[...slug]': './pages/shop/products/[...slug]',
  },
});
```

## 构建默认值

- client `publicPath` 会规范化为 `auto`
- server target 会规范化为 `async-node`
- remote 元数据来自 `mf-manifest.json`
