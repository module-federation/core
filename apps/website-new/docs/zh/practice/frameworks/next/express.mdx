# 使用 Express.js

如果你使用自定义 Express server 运行 Next，请以当前 `nextjs-mf` 合同为准：

- 仅支持 Pages Router
- 使用 `withFederatedRequest()`、`flushChunks()` 和 `FlushedChunks`
- 不要再从 `@module-federation/nextjs-mf/server` 导入 `revalidate()`

## 建议

把 federation 请求固定和资源注入放在 `pages/_document.js` 中处理，把 Express 自己的路由失效策略放在你的服务端代码里。

`nextjs-mf` 不再提供针对 Express 路由栈的进程级 revalidation hook。
