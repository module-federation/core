# React Bridge 概览

`@module-federation/bridge-react` 是一个强大的微前端解决方案，专为 React 应用设计。它允许您轻松地将 React 应用或组件作为独立模块进行导出和加载，同时提供智能的资源预加载和跨版本兼容能力。

## 核心功能

### 基础能力
- **🚀 应用级模块导出**：使用 `createBridgeComponent` 将完整的 React 应用包装为可远程加载的模块
- **📦 应用级模块加载**：通过 `createRemoteAppComponent` 在宿主应用中无缝集成远程 React 应用
- **🎯 组件级懒加载**：支持 `createLazyComponent` 实现细粒度的组件级懒加载

### 高级特性
- **⚡ 智能预加载**：内置资源预加载机制，支持应用、组件和依赖的预取优化
- **🔗 版本兼容**：完全支持 React 16-19，自动处理不同版本间的渲染差异和生命周期
- **🛣️ 路由无缝集成**：自动处理 React Router 的 basename 注入和路由上下文传递
- **� 渲染模式灵活**：支持标准渲染和内存路由模式，适配不同的应用场景

## 版本兼容支持

React Bridge 提供业界领先的版本兼容性，支持从 React 16 到 React 19 的所有主流版本：

| React 版本 | 渲染方式 | 支持状态 |
|-----------|---------|---------|
| **React 16.x** | `ReactDOM.render` | ✅ 完全支持 |
| **React 17.x** | `ReactDOM.render` | ✅ 完全支持 |
| **React 18.x** | `ReactDOM.createRoot` | ✅ 完全支持 |
| **React 19.x** | `ReactDOM.createRoot` | ✅ 完全支持 |

### 为什么这很重要？

- **渐进式升级**：允许不同团队按自己的节奏升级 React 版本
- **零中断集成**：新老版本应用可以无缝协作，无需同步升级
- **技术债务管理**：避免因版本不一致导致的集成问题
- **团队自主性**：每个团队可以独立选择最适合的 React 版本

## 智能版本适配

React Bridge 提供了灵活的导入方式，既支持自动检测，也支持显式指定版本：

### 自动检测（推荐）
```tsx
// 系统会自动检测当前 React 版本并使用对应实现
import { createBridgeComponent } from '@module-federation/bridge-react';
```

### 显式指定版本
```tsx
// 当需要明确指定版本时使用
import { createBridgeComponent } from '@module-federation/bridge-react/16'; // React 16/17
import { createBridgeComponent } from '@module-federation/bridge-react/18'; // React 18
import { createBridgeComponent } from '@module-federation/bridge-react/19'; // React 19
```

:::tip 最佳实践
推荐使用自动检测模式，它能确保使用最合适的实现，并在 React 版本升级时自动适配。
:::
