# Next.js

:::::danger Project Deprecation
Next.js 支持正在结束，[详情见此](https://github.com/module-federation/core/issues/3153)
:::::

`@module-federation/nextjs-mf` 现在是一个 rspack-first 的 Next 适配层，并且只保留了显式公开 API：

- `@module-federation/nextjs-mf`: Next+rspack federation 插件
- `@module-federation/nextjs-mf/server`: Pages Router SSR helpers

该包不再暴露通配 deep import、`./utils`，也不再支持旧的 `extraOptions` 参数。

## 适用范围

- Next `^12 || ^13 || ^14 || ^15 || ^16`
- Pages Router SSR
- 基于 rspack 的构建

第一阶段 **不包含** App Router 支持。

## 安装

请使用 rspack 版本的 Next 工具链，而不是依赖 webpack 本地环境变量技巧：

```bash
pnpm add next-rspack @rspack/core @module-federation/nextjs-mf
```

新的适配器不再要求 `NEXT_PRIVATE_LOCAL_WEBPACK`。

## 插件用法

根入口只接受标准 Module Federation 配置。

```js
const withRspack = require('next-rspack');
const NextFederationPlugin = require('@module-federation/nextjs-mf');

/** @type {import('next').NextConfig} */
const nextConfig = {
  webpack(config, options) {
    config.plugins.push(
      new NextFederationPlugin({
        name: 'home',
        filename: 'static/chunks/remoteEntry.js',
        remotes: {
          shop: `shop@http://localhost:3001/_next/static/${
            options.isServer ? 'ssr' : 'chunks'
          }/remoteEntry.js`,
        },
        exposes: {
          './menu': './components/menu',
          './pages/index': './pages/index',
        },
      }),
    );

    return config;
  },
};

module.exports = withRspack(nextConfig);
```

如果要做 Pages 路由拼接，请显式声明需要的页面 `exposes`。不要再依赖 `pages-map`、通配导出或 `exposePages`。

## Server 入口

`./server` 入口现在只保留很小的 SSR API：

- `withFederatedRequest()`
- `flushChunks()`
- `FlushedChunks`

请在 `pages/_document.js` 中使用它，让 SSR 资源收集保持请求级隔离，并以 manifest 为数据源：

```jsx
import Document, { Html, Head, Main, NextScript } from 'next/document';
import {
  FlushedChunks,
  flushChunks,
  withFederatedRequest,
} from '@module-federation/nextjs-mf/server';

class MyDocument extends Document {
  static async getInitialProps(ctx) {
    const { initialProps, chunks } = await withFederatedRequest(async () => {
      const initialProps = await Document.getInitialProps(ctx);
      const chunks = await flushChunks();

      return { initialProps, chunks };
    });

    return {
      ...initialProps,
      chunks,
    };
  }

  render() {
    return (
      <Html>
        <Head>
          <FlushedChunks chunks={this.props.chunks} />
        </Head>
        <body>
          <Main />
          <NextScript />
        </body>
      </Html>
    );
  }
}

export default MyDocument;
```

## 刷新模型

刷新逻辑现在完全由 `nextjs-mf` 自身处理：

- remote 元数据来自 `mf-manifest.json`
- 请求开始时固定当前 remote generation
- manifest 的 `buildVersion` 变化会为后续请求启用新 generation
- 已在执行中的请求会继续使用原来的 generation
- 旧 generation 会在引用计数清零后退休

`@module-federation/nextjs-mf/server` **不再导出** `revalidate()`。

## 资源解析

远程静态资源仍然会指向 remote 自己的 origin，但下面这两个旧开关已经不再属于支持 API：

- `enableImageLoaderFix`
- `enableUrlLoaderFix`

## Runtime Plugins

仍然可以使用自定义 runtime plugin：

```js
new NextFederationPlugin({
  runtimePlugins: [require.resolve('./runtime-plugin.js')],
});
```

## 导入 Remote

```js
import React, { lazy } from 'react';

const Menu = lazy(() => import('shop/menu'));
```

在 Pages Router 示例里，联邦懒加载边界优先使用 `React.lazy`，而不是 `next/dynamic`。
