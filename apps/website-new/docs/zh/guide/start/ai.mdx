# AI

## 让 AI 了解 MF 2.0

当前大多数 AI 模型的训练数据截止于 2024 年初，对 Module Federation 2.0 的了解非常有限甚至存在错误。以下资源可以帮助 AI 获取准确的 MF 2.0 知识。

### llms.txt

[module-federation.io](https://module-federation.io) 支持 [llms.txt](https://llmstxt.org) 标准，AI 系统可将其作为知识库使用。

你可以将以下链接直接提供给 AI Agent，让其在回答问题前先获取最新文档：

```
https://module-federation.io/llms.txt
```

### Markdown 文档

文档站点的所有页面均支持以 Markdown 格式访问 —— 将页面 URL 末尾的 `.html` 替换为 `.md` 即可获得纯文本版本，便于 AI 直接读取：

```
# 示例：将 HTML 版本
https://module-federation.io/guide/basic/runtime.html

# 替换为 Markdown 版本
https://module-federation.io/guide/basic/runtime.md
```

### 快速提示词

如果你只是临时询问 AI，可以在对话开头附上以下提示，让 AI 先查阅文档再作答：

```
在回答前，请先访问 https://module-federation.io/llms.txt 了解 Module Federation 2.0 的最新文档，然后再回答我的问题。
```

### mf-docs Skill

如果你使用 Claude Code、Cursor、Windsurf 等支持 Skills 的 AI 编辑器，可以安装 `mf-docs` Skill —— 它会在每次回答 MF 相关问题前自动从 `module-federation.io` 实时拉取最新文档，无需手动粘贴链接：

```bash
npx skills add module-federation/core --skill mf-docs -y
```

安装后，直接触发：

```
/mf-docs shared 里的 singleton 和 requiredVersion 有什么区别？
```

## Agent Skills

如果你使用 Claude Code、Cursor、Windsurf 等 AI 编辑器，可以安装 **MF Agent Skills** —— 一组预置的指令集，让 AI 具备处理 MF 2.0 专项任务的能力，包括：

- 实时拉取最新文档回答配置问题
- 将 Module Federation 一键接入已有项目
- 诊断类型错误、共享依赖冲突、配置异常
- 分析远端模块信息

👉 [查看 Agent Skills 详细说明](./skill)
