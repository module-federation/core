# Presets

## Shared

Next internals are still shared automatically by the adapter. You should not re-declare them unless you have a very specific reason:

- `next/dynamic`
- `next/head`
- `next/image`
- `next/link`
- `next/router`
- `next/script`
- `react`
- `react/`
- `react-dom`
- `react-dom/`
- `styled-jsx`
- `styled-jsx/style`

## Pages Exposes

The greenfield adapter does **not** support `extraOptions.exposePages`.

If another app imports a stitched Pages route, expose that page explicitly:

```js
new NextFederationPlugin({
  name: 'remoteApp',
  filename: 'static/chunks/remoteEntry.js',
  exposes: {
    './pages/index': './pages/index',
    './pages/shop/products/[...slug]': './pages/shop/products/[...slug]',
  },
});
```

## Build Defaults

- client `publicPath` is normalized to `auto`
- server target is normalized to `async-node`
- remote metadata is resolved from `mf-manifest.json`
