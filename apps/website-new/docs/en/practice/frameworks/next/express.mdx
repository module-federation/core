# Working with Express.js

If you run Next with a custom Express server, keep the current `nextjs-mf` contract in mind:

- Pages Router only
- use `withFederatedRequest()`, `flushChunks()`, and `FlushedChunks`
- do **not** use `revalidate()` from `@module-federation/nextjs-mf/server`

## Recommendation

Keep federation request pinning and asset flushing inside `pages/_document.js`, and keep any Express-specific route invalidation logic in your own server layer.

`nextjs-mf` no longer provides a process-wide revalidation hook for custom Express route stacks.
