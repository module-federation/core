# Angular Integration Overview

When integrating Module Federation into an Angular app, choose the guide based on the build setup used by your project.

| Project type | Read next |
| --- | --- |
| Angular CLI project | [Angular CLI Setup](/integrations/framework/angular/angular-cli.html) |
| Nx workspace | [Using Nx CLI for Angular](/integrations/framework/angular/using-nx-for-angular.html) |
| Full micro-frontend example | [Angular MFE](/integrations/framework/angular/angular-mfe.html) |
| SSR scenario | [Angular SSR](/integrations/framework/angular/mf-ssr-angular.html) |
| Service Worker scenario | [Service Workers with Module Federation](/integrations/framework/angular/service-workers-mf.html) |

Angular integrations commonly depend on Angular CLI, a custom Webpack builder, or Nx. Before choosing a guide, confirm your current build setup and whether SSR is required.

If you only need the shared Module Federation configuration concepts, see [Configuration](/configure/index.html) for `name`, `remotes`, `exposes`, and `shared`.
