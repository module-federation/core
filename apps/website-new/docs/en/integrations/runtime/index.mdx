# Runtime

If your project cannot use a build plugin, or you only need to register and load remote modules at runtime, start with the runtime-only approach.

Usually, use `@module-federation/enhanced/runtime` as the entry. For runtime APIs, module registration, module loading, and runtime plugins, continue with [Runtime](/guide/runtime/index).

## How to import Runtime APIs

If you need to call Runtime APIs such as `loadRemote`, `registerRemotes`, or `getInstance` manually, choose the import entry based on your integration:

| Integration | Runtime API import |
| --- | --- |
| Rspack / Webpack | Import from `@module-federation/enhanced/runtime` |
| Modern.js v3 | Import from `@module-federation/modern-js-v3/runtime` |
| Modern.js v2 | Import from `@module-federation/modern-js/runtime` |
| Rsbuild / Rslib / Vite / Rspress | Install `@module-federation/enhanced` separately and import from `@module-federation/enhanced/runtime` |

The Rsbuild, Rslib, Vite, and Rspress plugin packages do not provide a `runtime` subpath. If you only installed one of these plugin packages, do not import Runtime APIs from the plugin package.

This path is for special integration scenarios. Most web apps should start with the integration that matches the project, such as [Rsbuild](/integrations/build-tool/rsbuild), [Rspack](/integrations/bundler/rspack), [Webpack](/integrations/bundler/webpack), or [Modern.js](/integrations/framework/modernjs).
